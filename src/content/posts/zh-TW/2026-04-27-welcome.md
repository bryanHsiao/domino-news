歡迎！這是 **Domino News** 的第一篇文章。

本站專注於 [HCL Domino](https://www.hcl-software.com/domino) 與其周邊生態系（Notes、Domino REST API、Volt MX、Nomad、AppDev Pack、Sametime 等）的最新動態與技術深度文章。

## 為什麼要做這個站

HCL Domino 的生態系資訊很分散：HCL 官方、HCL Ambassador 部落格、planetlotus.org 聚合、OpenNTF 專案、GitHub release notes、開發者社群論壇 …… 中文圈尤其少。本站每天把這些散在各處的訊號整理成一篇好讀的繁中 + 英文文章，讓你開瀏覽器五分鐘就能掌握當日生態系動態。

## 運作方式：雙模型協作 + 人工把關

每日由 GitHub Actions 排程觸發三階段流程：

1. **OpenAI GPT 主筆草稿** —— 帶 web search 工具掃過去 24–72 小時的訊息，挑題、寫初稿、標 tag、列來源
2. **Anthropic Claude 交叉覆核** —— 跨廠商第二顆模型審稿，目的是降低單一模型的偏誤與幻覺；critical 等級的事實錯誤直接擋下重寫，不讓有問題的內容上線
3. **顧問實戰補充與校訂** —— 補上實務經驗、修掉 AI 容易誤標的 API 名稱、調整節奏與翻譯細節

當天若實在沒新聞可寫，會 fallback 到 TIER C：從 HCL 官方文檔挑一個沒寫過的 class 或 feature 寫深度教學（例如 [DQL 入門](/domino-news/posts/dql-getting-started)、[NotesViewNavigator 入門](/domino-news/posts/notes-view-navigator)）。

### 品質承諾

- 來源優先順序：HCL 官方頁 > HCL Ambassador / OpenNTF > 知名業務夥伴部落格（panagenda、prominic 等）
- 不引用無法驗證的內容；每篇文章末尾都會列出實際參考的來源連結
- 跨 LLM 廠商審稿 + 人工複核 是為了避開單純 AI 內容常見的問題：事實錯誤、虛構 API 名稱、來源誤標
- 嚴重事實錯誤無法修正時，當天寧可不發，也不發爛內容

## 預期主題

- 產品版本更新（release notes 整理）
- Volt MX、Nomad、Domino REST API 的開發實務與教學
- LotusScript / @Formula / Java 後端類別深掘
- 安全性與管理（admin、ID vault、HTTP 標頭、OIDC）
- 社群活動與技術分享
- AI 與 Domino 整合趨勢（如 Domino IQ）

## 整套工作流是 open source

包含產文 prompt、雙模型審稿邏輯、失敗草稿的人工救援機制等，[全部公開於 GitHub](https://github.com/bryanHsiao/domino-news)。歡迎參考、提 issue、或提出整套架構的改進建議。
