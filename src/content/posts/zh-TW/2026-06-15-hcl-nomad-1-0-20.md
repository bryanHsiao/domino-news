[HCL Nomad](https://help.hcl-software.com/nomad/welcome/index.html) 讓你不改寫程式、就把既有的 Notes / Domino 應用搬到瀏覽器與行動裝置上跑。截至 2026-06 寫稿時，Nomad for web browsers 與 Nomad server on Domino 的最新版本是 **1.0.20**。

這一版最值得 Domino 開發者注意的，是一個很實用、但容易被當成小功能略過的東西：**Kiosk Mode**。

## Kiosk Mode：把一個 Domino app 鎖成單一用途終端

[官方的 what's new](https://help.hcl-software.com/nomad/1.0_web/nomad_web_new.html) 對 Kiosk Mode 的描述是：可以指定**一個特定資料庫在開機時自動執行、且不允許被關閉**，其餘的 Nomad 介面元素也能透過設定隱藏或鎖定。

這解決的是一個很具體的場景：你有一個 Domino 應用 —— 報修登記、產線看板、接待櫃台簽到、活動報到機 —— 想把它擺在一台公用裝置上，讓使用者**只能用這一個 app、不能亂點別的、不能關掉視窗跑去開別的東西**。以前要靠瀏覽器或作業系統的 kiosk 設定東拼西湊，現在 Nomad 直接內建：指定那個 NSF、開機就進、鎖住，其他控制項照設定收掉。

對「想把舊的 Notes 應用變成一台專用機」的需求，這是第一次有官方、乾淨的做法。

## 另一個補丁：Firefox 的檔案匯出

1.0.20 也補上了 **Firefox 的檔案匯出支援** —— 在這之前，從 Nomad web 匯出檔案只在 Chrome / Edge 上可用，Firefox 使用者拿不到。這是個小但實在的相容性修補，把三大瀏覽器的行為拉齊。

## 放回脈絡：1.0.18 → 1.0.20 在做什麼

把最近幾版連起來看，會發現 Nomad web 這段時間有一條清楚的主線 —— **介面現代化（套用 HCL Enchanted 設計語言）+ 站穩部署場景**：

| 版本 | 寫稿時的時間點 | 重點 |
|---|---|---|
| 1.0.18 | 2025-12 | 改進 Nomad client 與 Nomad server on Domino 之間的登入流程；radio / checkbox / list box 等控制項套 Enchanted 設計 |
| 1.0.19 | 2026-03 | 更多控制項套 Enchanted：rich text lite、時間/時區、color picker、link hotspot 等 |
| **1.0.20** | 最新 | **Kiosk Mode**、Firefox 檔案匯出 |

換句話說，1.0.18、1.0.19 主要在「把畫面變好看、變一致」，1.0.20 則往「**怎麼把 Nomad 部署到真實場景**」推了一步 —— Kiosk Mode 正是部署面的功能，不是外觀。

## 升級前的小提醒

1.0.18 那次改了 client ↔ server on Domino 的登入流程，當時官方要求**各平台 client 一起更新**（web 1.0.17+、Android 1.0.72+、iOS 1.0.51+）才相容。如果你的環境還停在更舊的版本、要一路跳到 1.0.20，記得 client 跟 Nomad server on Domino 的版本要對得上，別只升一邊。實際的版本相依與修補清單，以[官方 release notes](https://help.hcl-software.com/nomad/1.0_web/nomad_web.html) 為準。

整體來說，1.0.20 不是大版本，但 Kiosk Mode 對特定部署場景是「有跟沒有差很多」的那種功能 —— 如果你正想把某個 Domino 應用變成公用終端機，這一版值得升。
