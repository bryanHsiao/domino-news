Welcome to **Domino News** — our first post.

This site focuses on [HCL Domino](https://www.hcl-software.com/domino) and its broader ecosystem: Notes, Domino REST API, Volt MX, Nomad, AppDev Pack, Sametime, and more — covering both the latest news and technical deep-dives.

## Why this site

HCL Domino's ecosystem signal is scattered across many places: HCL official pages, HCL Ambassador blogs, planetlotus.org aggregation, OpenNTF projects, GitHub release notes, the developer forums. Every day, this site pulls those signals into a single, well-edited post in both Traditional Chinese and English — five minutes of reading and you're caught up on the day.

## How it works: two-model collaboration plus a human checkpoint

A daily GitHub Actions cron triggers the three-stage pipeline:

1. **OpenAI GPT drafts the article** — with web search enabled, scans the last 24–72 hours, picks the topic, writes the first draft, tags it, and lists the sources.
2. **Anthropic Claude cross-reviews it** — a second model from a different vendor checks the draft. The point is to reduce single-model bias and hallucination; a critical factual issue blocks the publish and forces a rewrite, so problem content never reaches the site.
3. **The consultant adds hands-on context and corrections** — fills in real-world experience, fixes API names AI tends to mis-spell, tunes the pacing, and polishes translation details.

If there's genuinely no news on a given day, the pipeline falls back to TIER C — picking an under-covered class or feature from the official docs and writing a focused tutorial (e.g. [DQL: getting started](/domino-news/en/posts/dql-getting-started), [NotesViewNavigator](/domino-news/en/posts/notes-view-navigator)).

### Quality commitments

- Source priority: HCL official pages > HCL Ambassador / OpenNTF > known business-partner blogs (panagenda, prominic, etc.)
- No unverifiable content gets cited; every article ends with a list of the actual sources consulted
- The cross-LLM review plus human checkpoint exists specifically to catch what pure AI content gets wrong: factual mistakes, hallucinated API names, miscited sources
- When a critical factual error can't be repaired the same day, no post goes out — silence beats bad content

## What to expect
